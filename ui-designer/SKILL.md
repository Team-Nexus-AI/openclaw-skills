# UI Designer Skill

自动化 UI 设计与集成的完整工作流，从需求收集到代码落地一气呵成。

## 工作流（严格按顺序执行）

### Step 1：收集 UI 需求

向用户提问，收集以下信息（可一次性问完）：

- **目标页面**：要设计哪个页面/模块？（登录页、Dashboard、列表页等）
- **设计风格**：偏好什么风格？（现代简洁、Material Design、暗黑、品牌色等）
- **目标平台**：移动端、桌面端，还是响应式？
- **核心功能**：这个页面需要哪些关键交互元素？
- **参考案例**：有没有喜欢的参考设计？（可选）

收到回答后进入 Step 2。

### Step 2：阅读项目代码

读取项目结构，重点关注：

```
1. 查看项目根目录结构（ls / find）
2. 识别前端技术栈（package.json、框架类型）
3. 阅读现有 UI 相关代码（组件目录、样式文件、已有页面）
4. 了解路由结构（router 配置）
5. 识别已有的设计系统或组件库（Tailwind、Ant Design、MUI 等）
```

目标：理解技术栈约束，让生成的 UI 代码能无缝融入现有项目。

### Step 3：合成最终设计规格

结合 Step 1（用户需求）和 Step 2（代码分析），生成结构化的设计 Prompt。

参考 `references/stitch-prompt-guide.md` 构建高质量 Stitch Prompt。

向用户展示最终设计规格并确认后，进入 Step 4。

### Step 4：提供 Stitch Prompt 与操作指引

向用户输出以下内容：

**🎨 Google Stitch 设计 Prompt**

将 Step 3 生成的完整 Prompt 以代码块形式展示，方便用户一键复制。

**🔗 访问地址**

https://stitch.withgoogle.com（需要 Google 账号登录）

**📋 操作步骤**

1. 打开上方链接，登录 Google 账号
2. 将上方 Prompt 复制粘贴到输入框，回车生成
3. 等待约 30 秒，在画布上预览 UI 效果
4. 根据喜好用自然语言继续调整，例如：
   - `"改成深色模式，主色换成靛蓝色"`
   - `"底部加一个导航栏，包含首页、搜索、我的三个 tab"`
   - `"让整体更简洁，减少装饰性元素"`
5. 满意后点击右上角 **Export → Download Code**，下载 HTML/CSS/JS 压缩包
6. 将下载的文件（或代码内容）发给我，我来帮你融合到项目中

> 等待用户将下载的代码发过来后，进入 Step 5。

### Step 5：融合 UI 代码到项目

收到用户提供的 Stitch 导出代码后，根据 Step 2 识别的技术栈执行融合。

参考 `references/integration-guide.md` 获取各技术栈的具体融合方案。

**核心原则：**
- 保留项目已有的设计 token（颜色变量、字体、间距）
- 转换组件语法（HTML → React/Vue/Svelte 组件）
- 接入已有的路由和状态管理
- 保持代码风格一致（格式化、命名规范）

**完成后汇报：**
- 新增/修改了哪些文件
- 如何在项目中访问新页面
- 后续建议（接入真实数据、添加交互逻辑等）

## 注意事项

- Stitch 生成的是静态 UI 壳，不含业务逻辑，融合时需明确告知用户
- 若项目使用 TypeScript，融合时补充类型定义
- 若项目无前端代码（纯后端项目），在项目根目录创建 `ui/` 目录存放生成的 UI
